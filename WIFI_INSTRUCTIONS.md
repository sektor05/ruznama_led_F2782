# Инструкция по использованию WiFi точки доступа Ruznama Table

## Настройки точки доступа

- **SSID (имя сети)**: `ruznama_table`
- **Пароль**: отсутствует (открытая сеть)
- **IP адрес**: `192.168.4.1`
- **Порт веб-сервера**: `80`

## Как подключиться к веб-интерфейсу

### Шаг 1: Загрузка прошивки

```bash
pio run -t upload
```

### Шаг 2: Подключение к точке доступа

1. На вашем устройстве (телефон, ноутбук, планшет) откройте настройки WiFi
2. Найдите сеть с именем `ruznama_table`
3. Подключитесь к ней (пароль не требуется)

### Шаг 3: Открытие веб-интерфейса

В браузере введите один из следующих адресов:

- `http://192.168.4.1`
- `http://ruznama.local` (если поддерживается mDNS)

## Функции веб-интерфейса

- **Текущее время**: отображается в крупном формате вверху страницы
- **Дата**: показывается под временем
- **Расписание молитв**: список всех 6 молитв на текущий день
- **Следующая молитва**: подсвечивается фиолетовым цветом
- **Автообновление**: время обновляется каждую секунду автоматически

## API эндпоинты

### Получение текущего времени

**URL**: `http://192.168.4.1/time`

**Метод**: GET

**Ответ** (JSON):
```json
{
  "time": "12:45:30",
  "date": "16.02.2026",
  "hijri": {
    "day": 17,
    "month": 7,
    "year": 1445
  },
  "hijriOffset": 0
}
```

### Установка времени с телефона

**URL**: `http://192.168.4.1/settime`

**Метод**: POST

**Тело запроса** (JSON):
```json
{
  "year": 2026,
  "month": 2,
  "day": 16,
  "hour": 12,
  "minute": 45,
  "second": 30
}
```

**Ответ** (JSON):
```json
{
  "success": true,
  "message": "Время установлено успешно"
}
```

### Корректировка даты хиджры

**URL**: `http://192.168.4.1/hijri-offset`

**Метод**: POST

**Тело запроса** (JSON):
```json
{
  "offset": 1
}
```

**Ответ** (JSON):
```json
{
  "success": true,
  "offset": 1,
  "hijri": {
    "day": 18,
    "month": 7,
    "year": 1445
  }
}
```

### Обновление прошивки (.bin файл)

**URL**: `http://192.168.4.1/firmwareupdate`

**Метод**: POST (multipart/form-data)

**Параметры**:
- `firmware`: файл прошивки (.bin)

**Ответ** (JSON):
```json
{
  "success": true,
  "message": "Обновление завершено успешно"
}
```

### Получение статуса обновления прошивки

**URL**: `http://192.168.4.1/firmwareupdate/status`

**Метод**: GET

**Ответ** (JSON):
```json
{
  "status": "updating",
  "percent": 45
}
```

Статусы:
- `idle` - нет активного обновления
- `updating` - обновление в процессе
- `success` - обновление завершено успешно
- `error` - ошибка обновления

### Обновление расписания молитв (year_schedule.json)

**URL**: `http://192.168.4.1/ruznamaupdate`

**Метод**: POST (multipart/form-data)

**Параметры**:
- Файл: файл расписания (.json)

**Ответ** (JSON):
```json
{
  "success": true,
  "message": "Расписание успешно обновлено"
}
```

### Получение статуса обновления расписания

**URL**: `http://192.168.4.1/ruznamaupdate/status`

**Метод**: GET

**Ответ** (JSON):
```json

  "status": "updating",
```

Статусы:
- `idle` - нет активного обновления

- `success` - обновление завершено успешно

## Устранение неполадок

### Не удается подключиться к точке доступа

1. Убедитесь, что прошивка успешно загружена
2. Перезагрузите ESP32-S2 Mini (отключите и подключите питание)
3. Подождите 10-15 секунд после включения перед подключением
4. Проверьте Serial монитор на наличие ошибок

### Веб-страница не открывается

1. Убедитесь, что вы подключены к сети `ruznama_table`
2. Попробуйте ввести IP адрес вручную: `192.168.4.1`
3. Проверьте, что брандмауэр не блокирует соединение
4. Попробуйте другой браузер

### Расписание молитв не отображается

1. Проверьте Serial монитор на наличие ошибок инициализации файловой системы
2. Убедитесь, что файл `year_schedule.json` загружен в файловую систему
3. Проверьте, что RTC инициализирован корректно

## Мониторинг через Serial

Для отладки подключите ESP32-S2 Mini к компьютеру и откройте Serial монитор (скорость 9600 бод). Вы увидите:

- Сообщения об инициализации
- IP адрес точки доступа
- Текущее время
- Расписание молитв
- Информацию о следующей молитве

## Настройки (при необходимости)

Изменить настройки точки доступа можно в файле [`src/wifi_ap.h`](src/wifi_ap.h:1):

```cpp
#define WIFI_AP_SSID "ruznama_table"        // Имя сети
#define WIFI_AP_PASSWORD ""                 // Пароль (пустой = открытая сеть)
#define WIFI_AP_CHANNEL 1                   // Канал WiFi (1-13)
#define WIFI_AP_MAX_CONNECTIONS 4           // Максимальное количество подключений
#define WEB_SERVER_PORT 80                  // Порт веб-сервера
```

После изменения настроек перезагрузите прошивку:
```bash
pio run -t upload
```
