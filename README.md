# Ruznama Table ESP32-S2 Mini

Проект для ESP32-S2 Mini с функциональностью отображения времени молитв на основе расписания из JSON файла. Устройство отображает время, дату, расписание намазов и время до икамата на семисегментных индикаторах.

## Оборудование

- **ESP32-S2 Mini** - микроконтроллер
- **RTC DS3231** (ZS-042) - часы реального времени
- **74HC595** (10 шт.) - сдвиговые регистры для управления LED
- **Верхняя плата** - индикаторы времени и даты
- **Нижняя плата** - семисегментные индикаторы для расписания молитв

## Основные функции

### Отображение времени и даты
- Текущее время в формате ЧЧ:ММ:СС
- Дата в формате ГГГГ.ММ.ДД
- День недели (светодиоды)
- Автоматическое переключение между григорианским и хиджрским календарями (каждые 5 секунд)
- Месяц хиджры (светодиоды)

### Расписание молитв
- Отображение 6 молитв на семисегментных индикаторах:
  - Фаджр (утренняя молитва)
  - Восход
  - Зухр (полуденная молитва)
  - Аср (послеполуденная молитва)
  - Магриб (вечерняя молитва)
  - Иша (ночная молитва)
- Мигание времени молитвы при наступлении (1 минута)
- Отображение времени до икамата (7-я строка индикаторов)
  - Минуты до икамата (более 1 минуты)
  - Секунды до икамата (последняя минута с миганием)
  - Мигание "00" после икамата (1 минута)

### WiFi точка доступа
- Веб-интерфейс для управления устройством
- Синхронизация времени с телефона/компьютера
- Корректировка даты хиджры
- Captive Portal для автоматического перенаправления

## Установка и настройка

### 1. Установка зависимостей

Проект использует следующие библиотеки:
- `RTClib` - библиотека для работы с RTC DS3231
- `ArduinoJson` - библиотека для парсинга JSON файлов
- `LittleFS` - файловая система для ESP32

Все зависимости автоматически загружаются при сборке проекта.

### 2. Подключение плат

#### Верхняя плата (Top Board)

| ESP32-S2 Mini | Верхняя плата 74HC595 | Описание |
|---------------|----------------------|----------|
| GPIO13        | Пин 12 (RCK)         | LATCH (Storage Register Clock) |
| GPIO6         | Пин 11 (SRCK)        | CLOCK (Shift Register Clock) |
| GPIO8         | Пин 14 (SER)         | DATA (Serial Data Input) |
| 5V            | VCC                  | Питание |
| GND           | GND                  | Земля |

На верхней плате находятся:
- 5 регистров 74HC595 в цепочке
- Индикатор времени: `00:00:00`
- Индикатор даты: `0000.00.00/`
- Светодиоды дней недели (7 шт.)
- Светодиоды месяцев хиджры (12 шт.)

#### Нижняя плата (Bottom Board)

| ESP32-S2 Mini | Нижняя плата 74HC595 | Описание |
|---------------|---------------------|----------|
| GPIO12        | Пин 12 (RCK)        | LATCH (Storage Register Clock) |
| GPIO7         | Пин 11 (SRCK)       | CLOCK (Shift Register Clock) |
| GPIO9         | Пин 14 (SER)        | DATA (Serial Data Input) |
| 5V            | VCC                 | Питание |
| GND           | GND                 | Земля |

На нижней плате находятся:
- 5 регистров 74HC595 в цепочке
- 7 рядов семисегментных индикаторов для отображения времени молитв

#### RTC DS3231 (ZS-042)

| ESP32-S2 Mini | RTC ZS-042 | Описание |
|---------------|------------|----------|
| GPIO33        | SDA        | I2C Data |
| GPIO35        | SCL        | I2C Clock |
| 3.3V          | VCC        | Питание (логика) |
| GND           | GND        | Земля |

### 3. Загрузка файлов на файловую систему

Для загрузки файла расписания молитв на файловую систему ESP32 выполните команду:

```bash
pio run --target uploadfs
```

Эта команда загрузит все файлы из директории `data` на файловую систему LittleFS ESP32.

### 4. Загрузка прошивки

Для загрузки основной прошивки на ESP32 выполните команду:

```bash
pio run --target upload
```

Или для одновременной загрузки прошивки и файловой системы:

```bash
pio run --target upload --target uploadfs
```

| Команда | Описание |
|---------|----------|
| `SETTIME YYYY-MM-DD HH:MM:SS` | Установить дату и время |

## Функциональность

### PrayerSchedule

Класс `PrayerSchedule` предоставляет следующие методы:

- `begin()` - Инициализация файловой системы
- `loadSchedule(const String& date)` - Загрузка расписания на указанную дату
- `loadCurrentSchedule()` - Загрузка расписания на текущую дату
- `getPrayerTime(uint8_t index)` - Получение времени молитвы по индексу (0-5)
- `getPrayerTime(const String& name)` - Получение времени молитвы по названию
- `getNextPrayer(uint8_t hour, uint8_t minute)` - Получение следующей молитвы после указанного времени
- `getIqamahTime(uint8_t index)` - Получение времени икамата для молитвы по индексу
- `getMinutesToIqamah(uint8_t hour, uint8_t minute, uint8_t dayOfWeek)` - Получение времени до икамата в минутах
- `getSecondsToIqamah(uint8_t hour, uint8_t minute, uint8_t second, uint8_t dayOfWeek)` - Получение времени до икамата в секундах
- `getCurrentPrayerIndex(uint8_t hour, uint8_t minute)` - Получение индекса молитвы, время которой наступило
- `isLoaded()` - Проверка, загружено ли расписание

### RTC

Функции для работы с RTC:

- `rtcInit()` - Инициализация RTC
- `rtcGetTime()` - Получение текущего времени
- `rtcSetTime(DateTime dateTime)` - Установка времени
- `rtcSetTimeFromString(const char* timeString)` - Установка времени из строки "YYYY-MM-DD HH:MM:SS"
- `rtcSetTimeFromRussianFormat(const char* timeString)` - Установка времени из строки "DD.MM.YYYY HH:MM:SS"
- `rtcGetTemperature()` - Получение температуры
- `rtcPrintTime()` - Вывод текущего времени в Serial
- `rtcGetHijriDate()` - Получение даты хиджры
- `rtcSetHijriOffset(int offset)` - Установка смещения даты хиджры

## Мониторинг Serial

Для просмотра отладочной информации подключитесь к Serial монитору:

```bash
pio device monitor
```

Скорость Serial: 9600 бод

### Важное примечание о Serial мониторе

ESP32-S2 Mini использует USB-CDC для Serial соединения. При нажатии кнопки reset порт может временно исчезаться.

**Решение:** Программа ожидает подключения Serial монитора до 5 секунд после старта перед выводом сообщений. Это дает вам достаточно времени для подключения Serial монитора после reset.

Для лучшего результата:
1. Откройте Serial монитор (`pio device monitor`)
2. Нажмите кнопку reset на ESP32-S2 Mini
3. Подождите до 5 секунд - вы увидите все сообщения инициализации

## Известные проблемы

### ⚠️ Загрузка прошивки через WiFi не работает

В настоящее время функция обновления прошивки через веб-интерфейс (`/firmwareupdate`) не работает стабильно. Рекомендуется использовать USB для обновления прошивки.

### ⚠️ Загрузка расписания через WiFi не работает

В настоящее время функция обновления расписания молитв через веб-интерфейс (`/ruznamaupdate`) не работает стабильно. Рекомендуется использовать USB для загрузки файла расписания.

**Решение:** Используйте команду `pio run --target uploadfs` для загрузки файла расписания через USB.

## Примечания

- Файловая система LittleFS имеет ограниченный размер (обычно 1-4 MB)
- Убедитесь, что файлы не превышают доступное пространство
- После изменения файлов в директории `data`, необходимо снова выполнить команду `pio run --target uploadfs`
- Файл `year_schedule.json` имеет размер ~241 KB и содержит расписание на весь год
- Точка доступа WiFi поддерживает до 4 одновременных подключений
- Captive Portal автоматически перенаправляет устройства на главную страницу
