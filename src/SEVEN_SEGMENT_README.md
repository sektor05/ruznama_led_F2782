# Драйвер семисегментного индикатора

## Обзор

Драйвер `seven_segment` предназначен для управления семисегментными индикаторами на базе регистров 74HC595. Драйвер поддерживает две платы (верхнюю и нижнюю), каждая из которых содержит 5 регистров 74HC595 в цепочке.

## Архитектура

### Цепочка регистров 74HC595

Каждая плата имеет 5 регистров 74HC595, соединенных последовательно:

```
[ESP32] -> [1-й 74HC595] -> [2-й 74HC595] -> [3-й 74HC595] -> [4-й 74HC595] -> [5-й 74HC595]
            Цифра 1         Цифра 2         Ряды           Цифра 3         Цифра 4
```

### Порядок отправки данных

Данные в сдвиговые регистры заходят последовательно. Тот байт, что отправляется первым, "пролетает" через всю цепочку и оказывается в 5-й микросхеме.

Порядок в массиве данных (5 байт):

| Индекс | Микросхема | Функция |
|--------|-----------|---------|
| [0]    | 5-я       | Цифра 4 (единицы минут) |
| [1]    | 4-я       | Цифра 3 (десятки минут) |
| [2]    | 3-я       | Ряды (строки) |
| [3]    | 2-я       | Цифра 2 (единицы часов) |
| [4]    | 1-я       | Цифра 1 (десятки часов) |

## Формат времени

Время отображается в формате `HH:MM`:

```
[H1][H2] : [M1][M2]
 Ц1   Ц2     Ц3   Ц4
```

Где:
- H1, H2 — часы (десятки и единицы)
- M1, M2 — минуты (десятки и единицы)
- Ц1, Ц2, Ц3, Ц4 — цифры на семисегментных индикаторах

## Использование

### Инициализация

```cpp
#include "seven_segment.h"

void setup() {
    boardInit();
    sevenSegmentInit();
}
```

### Отображение данных (времени)

#### Отображение на верхней плате

```cpp
// Отобразить время 12:34 на ряду 1
displayDataTop(12, 34, 1);

// Отобразить время 23:59 на ряду 6
displayDataTop(23, 59, 6);
```

#### Отображение на нижней плате

```cpp
// Отобразить время 09:15 на ряду 3
displayDataBottom(9, 15, 3);
```

#### Универсальная функция

```cpp
// Отобразить время на указанной плате
displayData(BOARD_TOP, 12, 34, 1);    // Верхняя плата
displayData(BOARD_BOTTOM, 12, 34, 1); // Нижняя плата
```

### Очистка индикаторов

```cpp
// Очистить все индикаторы на верхней плате
clearDisplayTop();

// Очистить все индикаторы на нижней плате
clearDisplayBottom();

// Универсальная функция
clearDisplay(BOARD_TOP);
clearDisplay(BOARD_BOTTOM);
```

## Пример использования

### Пример 1: Отображение текущего времени

```cpp
#include "seven_segment.h"
#include "rtc.h"

void loop() {
    DateTime now = rtcGetTime();
    
    // Отобразить текущее время на ряду 1
    displayDataTop(now.hour(), now.minute(), 1);
    
    delay(1000);
}
```

### Пример 2: Отображение расписания молитв

```cpp
#include "seven_segment.h"
#include "prayer_schedule.h"

void updatePrayerDisplay() {
    const DailySchedule& schedule = prayerSchedule.getCurrentSchedule();
    
    // Отобразить каждую молитву на отдельном ряду
    for (int i = 0; i < 6; i++) {
        PrayerTime prayer = schedule.prayers[i];
        displayDataTop(prayer.hour, prayer.minute, i + 2);
    }
}
```

### Пример 3: Отображение времени до икамата

```cpp
#include "seven_segment.h"
#include "prayer_schedule.h"

void displayMinutesToIqamah(int minutes) {
    // Отобразить минуты до икамата в формате 00:MM
    displayDataTop(0, minutes, 1);
}
```

## Конфигурация

### Тип семисегментного индикатора

В файле `seven_segment.h` можно выбрать тип индикатора:

```cpp
// Общий анод (0 = включено)
#define SEGMENT_TYPE SEGMENT_TYPE_COMMON_ANODE

// Общий катод (1 = включено)
#define SEGMENT_TYPE SEGMENT_TYPE_COMMON_CATHODE
```

### Количество рядов

```cpp
#define NUM_ROWS 6  // Количество строк (рядов) на рузнама таблице
```

## Функции API

### Основные функции

| Функция | Описание |
|---------|----------|
| `sevenSegmentInit()` | Инициализация драйвера |
| `displayData(boardType, hours, minutes, row)` | Отображение данных (времени) |
| `displayDataTop(hours, minutes, row)` | Отображение данных (времени) на верхней плате |
| `displayDataBottom(hours, minutes, row)` | Отображение данных (времени) на нижней плате |
| `clearDisplay(boardType)` | Очистка индикаторов |
| `clearDisplayTop()` | Очистка индикаторов на верхней плате |
| `clearDisplayBottom()` | Очистка индикаторов на нижней плате |

### Низкоуровневые функции

| Функция | Описание |
|---------|----------|
| `shiftOutData(data, boardType)` | Отправка массива данных в регистры |

## Параметры

### Параметры функций отображения данных

| Параметр | Тип | Диапазон | Описание |
|----------|-----|----------|----------|
| `boardType` | `BoardType` | `BOARD_TOP`, `BOARD_BOTTOM` | Тип платы |

| `minutes` | `int` | 0-59 | Минуты |

## Схема подключения

### Пины ESP32-S2 Mini

| Функция | Пин GPIO | Описание |
|---------|----------|----------|
| `PIN_595_LATCH` | 7 | LATCH (Storage Register Clock) |
| `PIN_595_CLOCK` | 12 | CLOCK (Shift Register Clock) |
| `PIN_595_DATA_TOP` | 11 | DATA для верхней платы |
| `PIN_595_DATA_BOT` | 9 | DATA для нижней платы |

### Подключение к 74HC595

| Пин 74HC595 | Функция | Подключение |
|-------------|---------|-------------|
| 14 (DS) | Data Serial | ESP32 DATA pin |
| 12 (ST_CP) | Storage Register Clock | ESP32 LATCH pin |
| 11 (SH_CP) | Shift Register Clock | ESP32 CLOCK pin |
| 16 (VCC) | Питание | 5V |
| 8 (GND) | Земля | GND |
| 10 (MR) | Master Reset | VCC (через резистор) |
| 13 (OE) | Output Enable | GND |

## Устранение неполадок

### Индикаторы не отображают данные

1. Проверьте подключение пинов LATCH, CLOCK, DATA
2. Убедитесь, что питание 5V подключено правильно
3. Проверьте тип индикатора (общий анод/катод)
4. Проверьте правильность порядка отправки данных
5. Убедитесь, что вызывается функция `displayData()` или `displayDataTop()`/`displayDataBottom()`

### Отображаются неправильные цифры

1. Проверьте коды сегментов в `seven_segment.h`
2. Убедитесь, что выбран правильный тип индикатора
3. Проверьте подключение сегментов к регистрам

### Не работает определенный ряд

1. Проверьте функцию `getRowCode()` в `seven_segment.cpp`
2. Убедитесь, что микросхема управления рядами (3-я) подключена правильно
3. Проверьте маску активации рядов
