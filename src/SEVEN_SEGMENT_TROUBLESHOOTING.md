# Настройка драйвера семисегментного индикатора

## Проблемы с отображением

Если дисплей не работает корректно, попробуйте следующие настройки в файле [`seven_segment.h`](seven_segment.h).

## Настройка 1: Тип индикатора (общий анод/катод)

```cpp
// Выберите тип вашего индикатора:
#define SEGMENT_TYPE SEGMENT_TYPE_COMMON_ANODE   // Общий анод (по умолчанию)
// #define SEGMENT_TYPE SEGMENT_TYPE_COMMON_CATHODE  // Общий катод
```

**Как определить тип индикатора:**
- Общий анод: все аноды сегментов соединены вместе и подключены к VCC
- Общий катод: все катоды сегментов соединены вместе и подключены к GND

**Симптомы:**
- Если все сегменты постоянно горят или не горят вообще — попробуйте сменить тип индикатора
- Если отображаются перевернутые цифры — попробуйте сменить тип индикатора

## Настройка 2: Инверсия данных (если используется инвертор)

```cpp
// Если на дисплее ничего не отображается при логической 1 на DATA,
// установите это значение в 1
#define DATA_INVERTED 0  // Без инверсии (по умолчанию)
// #define DATA_INVERTED 1  // С инверсией
```

**Когда использовать:**
- Если на линии DATA логическая 1, но на дисплее ничего не отображается
- Если используется инвертор на линии данных (например, транзисторная сборка)

## Настройка 3: Порядок отправки битов

```cpp
// Попробуйте оба варианта, если дисплей не работает корректно
#define BIT_ORDER_LSBFIRST  0  // Младший бит первым (по умолчанию)
#define BIT_ORDER_MSBFIRST  1  // Старший бит первым

// Выберите порядок отправки битов:
#define BIT_ORDER BIT_ORDER_LSBFIRST  // LSBFIRST (по умолчанию)
// #define BIT_ORDER BIT_ORDER_MSBFIRST  // MSBFIRST
```

**Симптомы:**
- Если отображаются неправильные цифры (например, 1 отображается как 7)
- Если сегменты горят в неправильном порядке

## Порядок диагностики

### Шаг 1: Проверьте подключение

1. Проверьте подключение пинов LATCH, CLOCK, DATA
2. Убедитесь, что питание 5V подключено правильно
3. Проверьте землю (GND)

### Шаг 2: Проверьте тип индикатора

Попробуйте оба типа индикатора:
```cpp
#define SEGMENT_TYPE SEGMENT_TYPE_COMMON_ANODE   // Сначала попробуйте это
// #define SEGMENT_TYPE SEGMENT_TYPE_COMMON_CATHODE  // Затем попробуйте это
```

### Шаг 3: Проверьте инверсию данных

Если дисплей не отображает данные, попробуйте инверсию:
```cpp
#define DATA_INVERTED 0  // Сначала попробуйте это
// #define DATA_INVERTED 1  // Затем попробуйте это
```

### Шаг 4: Проверьте порядок битов

Если цифры отображаются неправильно, попробуйте изменить порядок:
```cpp
#define BIT_ORDER BIT_ORDER_LSBFIRST  // Сначала попробуйте это
// #define BIT_ORDER BIT_ORDER_MSBFIRST  // Затем попробуйте это
```

### Шаг 5: Проверьте задержки

Если дисплей работает нестабильно, попробуйте увеличить задержки в [`seven_segment.cpp`](seven_segment.cpp):

```cpp
void shiftOutData(uint8_t data[NUM_REGISTERS], BoardType boardType) {
    // ...
    LATCH_LOW();
    delayMicroseconds(1);  // Попробуйте увеличить до 5 или 10
    
    // ...
    
    LATCH_HIGH();
    delayMicroseconds(1);  // Попробуйте увеличить до 5 или 10
}
```

## Частые проблемы и решения

### Проблема: Все сегменты постоянно горят

**Решение:**
1. Попробуйте сменить тип индикатора (анод/катод)
2. Проверьте подключение общего провода индикатора (VCC или GND)

### Проблема: Все сегменты не горят

**Решение:**
1. Попробуйте сменить тип индикатора (анод/катод)
2. Проверьте питание 5V
3. Проверьте подключение общего провода индикатора (VCC или GND)
4. Попробуйте включить инверсию данных (`DATA_INVERTED 1`)

### Проблема: Отображаются неправильные цифры

**Решение:**
1. Попробуйте изменить порядок битов (`BIT_ORDER`)
2. Проверьте коды сегментов в [`seven_segment.h`](seven_segment.h)
3. Проверьте подключение сегментов к регистрам

### Проблема: Не работает определенный ряд

**Решение:**
1. Проверьте функцию `getRowCode()` в [`seven_segment.cpp`](seven_segment.cpp)
2. Убедитесь, что микросхема управления рядами (3-я) подключена правильно
3. Проверьте маску активации рядов

### Проблема: Дисплей работает нестабильно

**Решение:**
1. Увеличьте задержки в функции `shiftOutData()`
2. Проверьте длину проводов (слишком длинные провода могут вызывать помехи)
3. Добавьте конденсаторы 0.1 мкФ между VCC и GND около каждого регистра 74HC595

## Проверка осциллографом

### Проверка LATCH

- LATCH должен быть LOW во время отправки данных
- LATCH должен быть HIGH после отправки данных
- Импульс LATCH должен быть минимум 1 мкс

### Проверка CLOCK

- CLOCK должен иметь импульсы во время отправки данных
- Частота CLOCK должна быть в пределах от 1 кГц до 10 МГц
- Каждый байт данных должен сопровождаться 8 импульсами CLOCK

### Проверка DATA

- DATA должен изменяться в соответствии с отправляемыми данными
- DATA должен быть стабильным во время импульса CLOCK
- Если используется инвертор, DATA будет инвертирован относительно ожидаемого значения

## Контакты для связи

Если ни одна комбинация не работает, проверьте:
1. Схему подключения (возможно, используется нестандартная разводка)
2. Тип микросхем 74HC595 (возможно, используется 74HCT595 или другая серия)
3. Наличие дополнительных компонентов на линиях (инверторы, буферы и т.д.)
